How the software is split

This panel is built into the app so your hackathon team can explain and debug the architecture.

Frontend JavaScript

public/app.js reads forms, sends fetch() requests, and renders cards. public/sim.js parses and draws CAD/EDA assets in the browser.

Python backend

backend/main.py defines API routes. sourcing.py verifies source cards. safety.py blocks unsafe flows. instructions.py generates diagram data.

Data

data/store.json stores profiles, projects, and mock orders. It is simple on purpose so you can inspect it while debugging.

Debugging order

Open DevTools Console for browser errors. Use Network tab for API errors. Use the terminal for Python traceback messages.
